## 🏗️ What I'm Building

> La Trobe University, Melbourne. Back in Kanpur since 2009. Nearly two decades inside the **Akashdeep Group** — operations, accounts, hospital administration.
> Started coding again in 2026. Four months later: four live systems running at a real hospital.
> Using **Claude AI** as my pair programmer. Shipping one module at a time.

| Module | What It Does | Status | Stack |
|--------|-------------|--------|-------|
| **HRMS v3** | People, attendance, payroll, biometric | 🟢 Live — pilot | PHP · MariaDB · Vanilla JS |

## 🧠 How I Work

```
Sell what the client demands.
Build what makes sense.
Link everything quietly.
Ship what works. Don't wait for perfect.
```

- Every module is **standalone** — sells alone, works alone
- Every module is **quietly linked** — `org_id` consistent across all of them
- A client who buys HRMS today can add Inventory tomorrow — zero re-onboarding
- I use **AI as infrastructure** — Claude for architecture, code review, pair programming
